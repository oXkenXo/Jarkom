Konsep & Protokol Jaringan

1. HTTP menggunakan TCP Socket

Protokol HTTP berjalan di atas TCP (Transmission Control Protocol) yang menjamin pengiriman paket data secara andal, berurutan, dan bebas kesalahan (reliable). Digunakan untuk mengirim request dari Client, melewati Proxy, hingga ke Web Server.

2. QoS menggunakan UDP Socket

UDP (User Datagram Protocol) digunakan untuk pengujian kualitas jaringan (Quality of Service - QoS) karena tidak memiliki overhead handshaking TCP. Parameter yang diukur meliputi:

  • RTT (Delay): Waktu bolak-balik pengiriman paket.
  • Jitter: Variasi selisih delay antar paket berturut-turut.
  • Packet Loss: Persentase paket yang hilang selama pengiriman.
  • Throughput: Kecepatan transfer data riil dalam unit Kbps.

3. Proxy Caching (HIT / MISS)

Untuk mempercepat akses data, Proxy Server menyimpan halaman web yang sering diakses di dalam memori RAM (Dictionary). Ketika Client meminta halaman yang sama, Proxy mengirimkannya langsung dari memori (Cache HIT) tanpa menghubungi Web Server, menghemat bandwidth dan waktu.
